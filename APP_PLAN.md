# 🫱 Nudge (넛지) — 앱 기획안

> **한 줄 소개:** "딱 한 개만 해보자"를 실현하는 초미니멀 운동 카운터. 위젯 한 번 탭 = 운동 1회 기록.
>
> **작성일:** 2026-04-14
> **플랫폼:** iOS + watchOS (위젯 중심)
> **상태:** Phase 0 — 기획

## 🎯 기획 의도 (Why Nudge)

### 해결하려는 문제
- **맥스아웃과 반대 지점의 사용자**: 운동을 해야 한다는 마음은 있지만, "세트", "반복수", "주간 스케줄" 같은 단어 자체가 부담이 되어 시작 전에 포기하는 사람.
- 현태 본인 경험: "막상 한 개는 별것도 아니지" 하고 시작하면 몇 개 더 하고 싶어지는 게 인간 심리. 그런데 대부분의 앱은 처음부터 "목표 50회", "3세트 × 12회" 같은 걸 요구해서 그 문턱을 못 넘음.

### Nudge의 철학
- **"운동할 마음을 만드는 앱"이 아니라 "이미 한 개 한 마음을 배신하지 않는 앱"** 이 되는 것.
- 계획, 목표, 세트, 스케줄 = 전부 없음. **오직 "방금 한 개 했음"을 기록**하는 것만 존재.
- 행동경제학 "넛지"처럼, **부담 없는 한 번의 작은 행동이 다음 행동을 낳는다**는 전제에만 충실.

### 맥스아웃과의 포지셔닝
| 항목 | 맥스아웃 | Nudge |
|------|----------|-------|
| 대상 | 운동을 이미 시작한 사람 (점진적 과부하 추구) | 아직 운동 습관이 없는 사람 (시작 자체가 어려움) |
| 철학 | 최대치까지 짜낸다 | 딱 하나만 한다 |
| 복잡도 | 슈퍼세트·스마트 과부하·모션 카운팅 | 탭 한 번 = +1 |
| 기록 단위 | 세트/운동/주간 | 개별 횟수 (집계는 자동) |
| 주 UI | 앱 내 카운터 화면 | **홈 화면 위젯** |

두 앱은 사용자 여정의 다른 단계를 커버함. Nudge → 습관 형성 → 맥스아웃으로 넘어가는 자연스러운 사다리도 가능 (크로스 프로모션 여지).

## 📱 핵심 기능 (MVP)

### 1. 3가지 운동 고정
- **푸시업**
- **풀업** (집에 풀업바 있는 경우)
- **스쿼트**

언제 어디서든 맨몸으로 할 수 있다는 공통점으로 한정. 기구·헬스장 필요한 운동은 철학상 제외.

### 2. 앱 내 "오늘의 운동 선택"
- 앱에 들어가면 화면에 3가지 운동 중 하나를 선택하는 UI.
- 선택된 운동이 **현재 활성 운동**이 되고, 위젯이 이 운동을 기록함.
- 앱 내에서도 큰 "+1" 버튼으로 바로 기록 가능.

### 3. 홈 화면 위젯 (가장 중요)
- 위젯을 **한 번 탭하면 활성 운동의 오늘 횟수가 +1**.
- 위젯에는 표시되는 것:
  - 활성 운동 이름 (푸시업 / 풀업 / 스쿼트)
  - 오늘 누적 횟수 (큰 숫자)
  - 은은한 "탭하면 +1" 힌트 (iOS 위젯 인터랙션 가이드라인 내에서)
- 중형/소형 위젯 두 사이즈 제공.

### 4. Apple Watch 앱
- 워치페이스/컴플리케이션에서도 탭으로 +1 가능.
- 운동 중에도 손목을 까딱 → 워치에서 탭 → 기록.

### 5. 통계 화면 (성취감 부여)
- **일간**: 오늘 운동별 횟수 + 간단한 합계
- **주간**: 최근 7일 그래프 (운동별 스택 바)
- **월간**: 월 누적 + 일별 히트맵 (GitHub contribution 스타일)
- **연간**: 연 누적 + 월별 추세
- **평생 누적**: 이 앱을 쓴 이후 총 몇 개 했는지 (허영 지표, 하지만 성취감 제공)

### 6. 뭐가 *없는가* (의도적 제외)
- ❌ 목표/목표치 설정
- ❌ 세트/회차 개념
- ❌ 주간 스케줄, 리마인더 알림 (습관 강요 금지)
- ❌ 폼/자세 체크
- ❌ 타이머/휴식 시간
- ❌ 소셜/랭킹 (비교로 인한 부담 제거)
- ❌ 프리미엄 결제, 구독 (MVP 단계에선 고려 안 함)

## 🎨 UX 원칙

1. **한 동작으로 끝난다** — 위젯 탭 1회 → 기록 완료. 확인 다이얼로그 없음.
2. **실수 회복 경로는 제공** — 앱 내에서 오늘 기록 1개 취소 가능 (-1 버튼). 위젯에는 없음 (실수 방지보다 마찰 없는 기록이 우선).
3. **칭찬 최소화, 숫자 증가만** — "잘했어요!" 팝업 같은 거 없음. 숫자가 올라가는 것 자체가 보상.
4. **색/타이포로만 감정 전달** — 탭하면 미세한 햅틱 + 숫자 카운트업 애니메이션.
5. **처음 여는 사람에게 절대 튜토리얼 강요 안 함** — 운동 고르기 → 바로 "+1" 가능한 화면.

## 🏗️ 기술 스택 (초안)

| 영역 | 선택 | 이유 |
|------|------|------|
| 언어/프레임워크 | SwiftUI | 맥스아웃과 스택 통일, 재사용성 |
| 위젯 | WidgetKit + App Intents | iOS 17+부터 위젯 탭으로 앱 코드 실행 가능 (Interactive Widget) |
| 데이터 저장 | SwiftData + App Group | 앱과 위젯이 동일 DB 공유 |
| 클라우드 동기화 | CloudKit | iPhone ↔ Watch ↔ (나중에) iPad 동기화. 무료. |
| Watch 통신 | WatchConnectivity | 실시간 동기화 필요 없고 CloudKit으로 충분하면 생략 가능 |
| 통계 차트 | Swift Charts | iOS 16+ 기본 제공 |

### 핵심 기술 챌린지
- **Interactive Widget + App Intents**: 위젯 탭으로 카운트 증가시키는 건 iOS 17부터만 됨. 최소 타겟은 iOS 17.
- **App Group 설정**: 앱과 위젯이 SwiftData 공유하려면 App Group 활성화 필수.
- **위젯 새로고침 타이밍**: 탭 직후 숫자가 바로 반영되도록 `WidgetCenter.shared.reloadAllTimelines()` 적절히 호출.

## 💡 추후 고려 (MVP 이후)

- **Live Activity / Dynamic Island**: 운동 중 (예: 스쿼트 10개 하는 중) 다이나믹 아일랜드에 실시간 표시. 탭하면 +1.
- **잠금 화면 위젯**: iOS 16+ 잠금 화면에서도 탭 가능한지 검증.
- **운동 추가**: 버피, 런지 등 사용자가 요청하면 추가. 단, **한 번에 3개만 고정 노출** 원칙 유지 (미니멀리즘).
- **HealthKit 연동**: 기록을 건강 앱에 반영 (선택 사항).
- **맥스아웃 연동**: Nudge 사용자 중 일정 시점부터 맥스아웃 앱을 추천 (사용자 여정 설계).
