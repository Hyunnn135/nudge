# 🎨 Nudge — 디자인 시스템 초안

> 작성일: 2026-04-14 · Phase 0 마지막 체크박스
> 목적: 아이콘/컬러/타이포 컨셉을 문서화해 Phase 1 Xcode 셋업으로 매끄럽게 넘어가기 위함

---

## 1. 디자인 원칙

1. **여백이 곧 UI다** — 버튼·숫자·아이콘 주변에 숨 쉴 공간을 크게 둔다. 압박감 제로가 철학의 시각화다.
2. **정보는 숫자 한 개만** — "오늘 45개"처럼 단일 숫자가 화면 중앙에서 주인공이다. 부가 지표는 2차 정보 위계로만.
3. **색은 한 가지 감정만 말한다** — 민트/티얼은 **차분한 신뢰**. 경고·보상 톤을 섞지 않는다.
4. **애니메이션은 반응을 위한 것** — 탭 → 카운트업 0.25s 이외의 연출 없음.
5. **맥스아웃과 반대편을 고수** — 맥스아웃이 강렬한 주황·대비·근육감이면, Nudge는 부드러운 민트·여백·추상.

## 2. 컬러 팔레트

### 2.1 프라이머리 (Teal)

| 이름 | Hex | 용도 |
|------|-----|------|
| `teal/50`  | `#f0fdfa` | 앱 배경 (라이트 모드) |
| `teal/100` | `#ccfbf1` | 카드 배경·비활성 상태 |
| `teal/300` | `#5eead4` | 프로그레스 바·서브 포인트 |
| `teal/500` | `#14b8a6` | **프라이머리 — 버튼/위젯 배경/아이콘** |
| `teal/600` | `#0d9488` | Pressed state·그라데이션 하단 |
| `teal/700` | `#0f766e` | 다크 모드 포인트 |

그라데이션 한 쌍 (위젯 배경 후보): `linear-gradient(135deg, #14b8a6, #0d9488)`

| 이름 | Hex | 용도 |
|------|-----|------|
| `ink`       | `#0f172a` | 주요 텍스트 (라이트) |
| `slate/600` | `#475569` | 보조 텍스트 |
| `slate/400` | `#94a3b8` | Placeholder·비활성 |
| `slate/200` | `#e2e8f0` | 구분선 |

### 2.3 다크 모드 맵핑

| 역할 | 값 |
|------|-----|
| 배경 | `#0a1010` (거의 검정, 약간 녹기 섞음) |
| 카드 | `#111c1a` |
| 프라이머리 | `#2dd4bf` (teal/400, 라이트보다 한 단계 밝게) |
| 주요 텍스트 | `#e2f5f1` |

### 2.4 맥스아웃과의 컬러 대비

| 항목 | 맥스아웃 | Nudge |
|------|----------|-------|
| 메인 | `#7c2d12` (주황 고동) | `#14b8a6` (민트 티얼) |
| 감정 | 각성·의지·불꽃 | 평온·시작·틈 |
| 온도 | 따뜻한 붉은 톤 | 차가운 녹·청록 |

→ 두 앱을 동일 개발자 포트폴리오에 나란히 두었을 때 **한눈에 "정반대 페르소나"** 로 읽혀야 한다.

## 3. 타이포그래피

| 역할 | 폰트 | 웨이트 | 크기 (iOS) |
|------|------|--------|-----------|
| 카운터 대형 숫자 | SF Pro Rounded | Bold | 96~120pt |
| 화면 타이틀 | SF Pro | Semibold | 28pt |
| 본문 | SF Pro | Regular | 17pt |
| 부가 메타 | SF Pro | Medium | 13pt |
| 숫자 (위젯 소형) | SF Pro Rounded | Bold | 48~56pt |
| 숫자 (위젯 중형) | SF Pro Rounded | Bold | 72pt |

**결정 근거:** Rounded는 "부담 낮춤" 어조와 일치. 날카로운 서체는 넛지 철학과 충돌한다.

- **코너 반경:** `card` 20pt · `button` 14pt · `widget-tile` 22pt

- **그림자:** 거의 없음. 필요 시 `0 2px 12px rgba(20, 184, 166, 0.08)` 한 단계만.

## 5. 아이콘 컨셉 4종 (후보)

상세 SVG: `~/Desktop/Projects/nudge/icon-concepts/` 에 4개 파일로 보관. 모든 컨셉은 1024×1024 캔버스 기준이며 iOS 마스크(둥근 사각)가 적용될 것을 가정한다.

| # | 이름 | 표현 | 강점 | 약점 |
|---|------|------|------|------|
| A | **Ripple Tap** | 중앙 솔리드 원 + 바깥쪽으로 퍼지는 얇은 링 1개 | 탭의 파동 = "작은 행동이 번진다"는 철학을 그대로 시각화 | 유사 명상/알람 앱 아이콘과 혼동 가능 |
| B | **+1 Badge** | 큰 "+1" 타이포 (SF Rounded Bold) + 민트 배경 | 앱이 하는 일이 즉각 드러남. 스토어에서 기능 전달 가장 빠름 | "너무 명시적" — 미니멀 정체성과 살짝 어긋날 수 있음 |
| C | **Single Notch** | 흰 바탕 + 중앙 세로 짧은 획 하나 (tally mark의 첫 획) | 극단적 미니멀. "한 개" 라는 의미와 직결 | 가독성 테스트 필요 — 멀리서 "ㅣ" 로만 보일 위험 |
| D | **Fingertip Dot** | 둥근 모서리 사각 안에 작은 원 하나를 살짝 위쪽에 배치 | 여백 미학 극대화. 본능적으로 "누르고 싶다" 유발 | 심볼이 작아 홈스크린에서 존재감 약할 수 있음 |

### 추천 선택 순위

1. **A (Ripple Tap)** — 철학·시각 둘 다 강한 1순위. 작은 행동이 번지는 은유가 단순한 도형으로 구현됨.
2. **B (+1 Badge)** — 스토어 전환율 관점에서 보험. A가 약하면 백업.
3. **D (Fingertip Dot)** — 정체성은 최강이나 기능 전달이 약해 보조 후보.
4. **C (Single Notch)** — 실험적. 마케팅용 이미지엔 매력적이지만 앱 아이콘으로는 리스크.

→ **Phase 1 들어가기 전 A안으로 가기로 가결정. B는 A/B 테스트용 대안으로 보관.**

## 6. 위젯 비주얼 초안 (텍스트 스펙)

### 소형 위젯 (2×2)
```
┌────────────────────┐
│  PUSH-UP           │  ← teal/500 10pt, 좌상단
│                    │
│      45            │  ← SF Rounded Bold 56pt, 중앙
│                    │
│   tap to +1        │  ← slate/400 11pt, 하단
└────────────────────┘
배경: white (라이트) / #111c1a (다크)
테두리: 없음, 코너 22pt
```
