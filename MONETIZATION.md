# 💰 Nudge — 수익화 전략 (v2)

> 작성: 2026-04-14 (v1) · 수정: 2026-04-15 (v2, 전면 개정)
> **현재 정책: 완전 무료 + 선택적 $0.99 팁잘. 광고·구독·기능 잠금 없음. 영구.**
> 본격 수익화는 자매 앱 **맥스아웃**(구독제)이 담당하고, Nudge는 그 유입 깔때기 역할에 집중.

## 2. 왜 이 방향인가

### 2.1 Nudge는 맥스아웃의 유입 깔때기
- Nudge의 출발 자체가 "운동 부담이 큰 사람"을 운동 습관권으로 끌어오는 앱. 습관이 잡히면 자연스럽게 "더 체계적으로 하고 싶다" → 맥스아웃으로 넘어가는 사용자 여정.
- 즉 Nudge 자체의 수익화는 애초에 이 앱의 목적이 아니다. **유입 경로에서 마찰을 만들면 안 된다**는 게 제1원칙.
- 실질 수익은 **맥스아웃 구독제**로 잡고, Nudge는 최대한 깨끗하게 운영해 스토어 별점·리뷰·크로스 프로모션 자산을 쌓는다.

### 2.2 팁잘 방식이 맞는 이유
- **관계가 달라진다.** "광고 제거해줄 테니 돈 내라"는 압박 구조와 "좋았으면 커피 한 잔"은 근본적으로 다른 심리 계약. 후자는 사용자 만족도·리텐션에 플러스로 작용.
- **미니멀 정체성 그대로 유지.** 팁 버튼은 설정 화면 하단에 카드 하나로만 존재. 메인 UX 어디에도 노출 안 됨.
- **가격 $0.99의 의미.** 심리적 저항선이 가장 낮은 가격. "이 정도면 부담 없다"로 읽히는 구간. 주 수익이 목적이 아니라 **팬층 형성**이 목적.
- **Consumable로 설정** — 같은 사용자가 여러 번 후원할 수 있음. Non-consumable로 하면 한 번 사고 끝이라 "기여"의 성격이 희석됨.

### 2.3 구독을 뺀 이유
- 구독은 "매달 부담"이라는 압박을 만든다. Nudge 철학(계획·목표·스케줄 없음)과 정면 충돌.
- Nudge에서 구독을 제공할 만한 "프리미엄 기능" 자체를 설계하지 않는 것이 본 앱의 핵심. 그런 기능이 생기는 순간 Nudge는 이미 Nudge가 아니다.
- 구독제가 맞는 자리는 **맥스아웃**. 거기서 슈퍼세트 플랜·고급 통계·AI 자세 분석 같은 구독 가치를 설계.

## 3. Nudge가 수익화에서 까다로운 이유 (제약 조건 · v1에서 유지)

본 결정을 뒷받침하는 구조적 분석. 이 제약들이 "팁잘만으로 충분"이라는 결론의 근거.

### 3.1 Apple 정책 — 위젯 내부 광고 금지
- WidgetKit 위젯은 광고 불가. Nudge는 위젯이 메인 UI라 광고 활용 공간이 원천적으로 좁다.

### 3.2 짧은 세션 길이
- 들어와서 +1 누르고 나가는 패턴. 평균 세션 5~10초. 광고 노출 구조가 성립 안 함.

### 3.3 미니멀 정체성과의 구조적 충돌
- "계획·목표·알림 다 없애는 앱"인데 광고·구독·기능 잠금은 모두 **추가 마찰**. 정체성 훼손 비용이 수익보다 크다.

### 3.4 iOS ATT 이후 광고 단가
- 소규모 앱은 eCPM $0.5~$2 수준. Nudge 규모에서 광고 연간 수익이 팁잘 1건 팔리는 수익과 비슷해질 위험이 있음.

## 4. 팁잘 구현 가이드 (Phase 1~2 중 구현)

### 4.1 StoreKit 2 상품 설계
```
Product ID: site.salarykorea.nudge.tip.coffee
Type: Consumable In-App Purchase
Display Name: 개발자에게 커피 한 잔
Description: 좋았다면 커피값으로 한 잔 응원해주세요. 기능엔 아무 변화도 없어요.
Price Tier: Tier 1 ($0.99 / ₩1,400)
```

### 4.2 App Store Connect 등록 절차
1. App Store Connect → 앱 선택 → "기능(Features)" → "앱 내 결제"
2. "+" → "소모성(Consumable)" 선택
3. 참조 이름: `Coffee Tip`, 제품 ID: 위 값, 가격 티어 선택
4. 현지화된 이름/설명 한국어+영어 모두 입력
5. 리뷰 스크린샷: 팁 버튼이 보이는 설정 화면 캡처 업로드
6. "심사 준비" → 앱 빌드와 함께 심사 제출

### 4.3 SwiftUI 통합 개요

```swift
import StoreKit

@MainActor
final class TipJar: ObservableObject {
    static let productID = "site.salarykorea.nudge.tip.coffee"
    @Published var product: Product?
    @Published var purchasing = false
    @Published var thankYou = false
    
    func load() async {
        let products = try? await Product.products(for: [Self.productID])
        product = products?.first
    }
    
    func buy() async {
        guard let product else { return }
        purchasing = true
        defer { purchasing = false }
        let result = try? await product.purchase()
        if case .success(let verification) = result,
           case .verified(let tx) = verification {
            await tx.finish()
            thankYou = true
        }
    }
}
```

### 4.4 UI 배치 (설정 화면 하단 카드)
- 톤: "개발자에게 커피 한 잔 사주세요 ☕️"
- 작은 서브텍스트: "기능엔 아무 변화도 없어요. 그냥 응원이에요."
- 버튼: "$0.99 후원하기"
- 구매 성공 시: 하트 이모지 애니메이션 + "감사합니다!" 토스트 2초 표시 후 닫힘
- 재구매 가능(consumable), 기록 남기지 않음 (익명성 유지 / 사용자 입장에선 "몇 번 후원했는지" 추적되는 게 오히려 부담)

### 4.5 테스트
- Xcode → Sandbox 계정으로 결제 플로우 테스트
- 실제 심사 제출 전 StoreKit Testing(Xcode 로컬 `.storekit` 파일)로 유닛 테스트 가능

## 5. 맥스아웃과의 연결 (크로스 프로모션)

Nudge를 유입 깔때기로 활용하는 구체적 전략:

- **사용 패턴이 성숙한 시점**에만 맥스아웃을 부드럽게 소개. 예: Nudge 30일 연속 사용 후 설정 화면에 "더 깊게 가고 싶다면..." 카드 한 번 노출.
- 절대 **푸시 알림/팝업/강제 튜토리얼**로 유도 금지. Nudge 철학 위반.
- 맥스아웃 구독 정책은 별도 문서(`maxout/MONETIZATION.md` — 미작성)에서 관리. Nudge에서는 링크만.
- Nudge 사용자에게 **맥스아웃 첫 달 구독 할인** 쿠폰 제공 같은 장치는 Phase 4 이후에 검토.

## 6. 이전 결정(v1, 2026-04-14) 기각 사유 아카이브

v1에서는 Freemium 하이브리드 3단계(v1.0 무광고 → v1.1 광고제거 IAP $2.99 → v1.2 통계 interstitial)를 제안했으나 **2026-04-15 전면 기각**.

**기각 사유:**
1. "광고 제거 IAP"는 순수 기부가 아니라 **불편 해소 구매**가 되어 사용자 관계가 거래적(transactional)으로 변한다. Nudge의 팬층 형성 전략과 상충.
2. Nudge가 맥스아웃의 유입 깔때기임을 고려하면, Nudge 자체에서 수익 구조를 복잡화할 이유가 없다. **단순성이 곧 유입 효율**.
3. v1.2의 interstitial 도입은 별점 유지 리스크가 있는데, 별점이 낮아지면 맥스아웃 유입도 함께 감소한다. 즉 본체(맥스아웃) 수익에까지 간접 타격.

**v1에서 가져온 것:**
- "광고는 위젯/메인 카운터/운동 선택 화면에 절대 금지" — 여전히 유효. 단, v2에서는 광고 자체를 안 넣으므로 사실상 사문화.
- "구독은 Nudge와 부정합" — 그대로 유지, 오히려 더 강하게 못박음.
